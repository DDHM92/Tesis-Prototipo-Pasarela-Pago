Introducción

Este prototipo de pasarela de pago multicomercio fue desarrollado como parte del proyecto de graduación para la Maestría en Arquitectura de Software. Su propósito principal es ofrecer una plataforma centralizada que permita a múltiples comercios gestionar sus pagos, ventas y envíos de forma eficiente, segura y personalizada.

Características

  • Autenticación multifactor para mayor seguridad.
  • Integración con múltiples métodos de pago como tarjetas de crédito y débito.
  • Automatización de tareas como recordatorios de pago y resúmenes de transacciones.
  • Diseño adaptable e intuitivo para usuarios con distintos niveles de experiencia.

Objetivos

El objetivo principal del prototipo es demostrar cómo una pasarela de pago puede satisfacer las necesidades de un entorno multicomercio mediante un diseño centrado en la usabilidad, la seguridad y la escalabilidad. Además, busca:

  • Facilitar la integración de nuevos comercios.
  • Proveer reportes detallados para la toma de decisiones.
  • Garantizar transacciones seguras y encriptadas.

Tecnologías Utilizadas

Este prototipo fue desarrollado utilizando tecnologías modernas para garantizar su funcionalidad y escalabilidad. Algunas de ellas incluyen:

  • Lenguaje de programación: C#
  • Framework: ASP.NET WebForms
  • Base de datos: SQL Server
  • Diseño Front-End: Bootstrap
